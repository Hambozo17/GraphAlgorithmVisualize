üî¨ Implemented Algorithms

Graph Traversal

Breadth-First Search (BFS)

Explores level by level using a queue

Time: O(V + E)

Depth-First Search (DFS)

Explores as deep as possible before backtracking

Time: O(V + E)

Shortest Path

Dijkstra's Algorithm

Greedy approach for non-negative weights

Time: O((V + E) log V)

Bellman-Ford Algorithm

Handles negative weights, detects cycles

Time: O(V √ó E)

A* Search Algorithm

Heuristic-guided optimal pathfinding

Time: O(E) to O(b^d)

üîç BFS vs DFS

Breadth-First Search

  • Uses Queue (FIFO)
  • Explores neighbors first
  • Finds shortest path in unweighted graphs
  • Level-order traversal

Applications:

  • Social network analysis
  • Web crawling
  • GPS navigation

Depth-First Search

  • Uses Stack (LIFO)
  • Goes deep before backtracking
  • Memory efficient
  • Can detect cycles

Applications:

  • Topological sorting
  • Maze solving
  • Cycle detection

üìç Shortest Path Algorithms

Dijkstra's Algorithm

  • Greedy approach
  • Uses priority queue
  • Only non-negative weights
  • Optimal for most cases
dist[v] = min(dist[v], dist[u] + w(u,v))

Bellman-Ford Algorithm

  • Dynamic programming approach
  • Handles negative weights
  • Detects negative cycles
  • Relaxes edges V-1 times
Relax all edges |V| - 1 times

‚≠ê A* Search Algorithm

f(n) = g(n) + h(n)

g(n)

Actual cost from start to node n

h(n)

Heuristic estimate to goal (Euclidean distance)

f(n)

Total estimated cost through n

Why A*?

  • Combines best of Dijkstra and greedy search
  • Optimal with admissible heuristic
  • More efficient than Dijkstra for goal-directed search
